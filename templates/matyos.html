§ 1

Quick Start

MatyOS programs are written in the El (Elnamic) language with the .el file extension. Every program begins with an ALGORITHM block. Write a file, run it, then write your first formal proof — in three steps.

Step 01 · Write
Hello World
hello_world.el 
ALGORITHM hello_world {
    show "Hello, MatyOS!";
}
Step 02 · Run
Execute
terminal 
el run hello_world.el

# Output:
Hello, MatyOS!
Step 03 · Prove
First Formal Proof
proof.el 
axiom id: true === true;
theorem t: true;
proof t {
    hypothesis h: true;
    test v: h: true;
    QED;
}

§ 2

Language Reference

The El language is statically typed with explicit variable declarations and supports five primitive types including the unique realistic three-valued boolean. Keywords are case-insensitive.

Variable & constant declarations 
// Declare with type
var x, y     : integer;
var s        : real;
var name     : string = "El";
var flag     : boolean = true;
var state    : realistic;

// Constants (immutable)
const PI     : real    = 3.14159;
const MAX_N  : integer = 1000;

// Multiple same-type declarations
var a, b, c  : string = "same";

// Arithmetic assignment
x = 1 * 2 + 3 - 4 / 5 * (1 + 2);

// Boolean expressions
flag = true and false or !false;
flag = x < 5 and y > 0;
TypeDescriptionExample literalNotes
integer Whole number (signed) 42, -7 Default numeric type
real Floating-point number 3.14, -0.5 Also aliased as float
string Text, double-quoted "hello" Supports trim(), upper()
boolean Two-valued logic true, false Operators: and, or, !
realistic Three-valued Kleene logic true | false | realistic ★ unique to El

The realistic type extends boolean with a third truth value representing uncertainty or superposition — implementing Kleene's K3 three-valued logic natively in the type system.

if / elif / else, for, while, do-while, switch 
// Conditional
if x > 5 {
    show "greater";
} elif x < 5 {
    show "less";
} else {
    show "equal";
}

// For loop (C-style)
for i: integer = 0; i < 10; i = i + 1 {
    show i;
}

// While loop
while counter <= 10 do {
    show counter;
    counter = counter + 1;
}

// Do-while loop
do {
    show x;
    x = x + 1;
} while x < 5

// Switch statement
switch(value) {
    case 1: show "one";
    case 2: show "two";
    default: show "other";
}
Function declarations, nesting, recursion 
// Basic function
function add(a: integer, b: integer) {
    return a + b;
}

// Nested function
function outer(s: string) {
    var c : string;
    function inner() {
        return 42;
    }
    return inner();
}

// Recursive function
function fibonacci(n: integer) {
    if n <= 1 { return n; }
    return fibonacci(n - 1) + fibonacci(n - 2);
}

// Call it
show fibonacci(10);   // → 55
show add(3, 4);       // → 7
Array declaration and iteration 
// Declare typed array
var numbers: array[integer] = [1, 2, 3, 4, 5];
var names:   array[string]  = ["El", "Lean", "Coq"];

// Iterate with for-in
var total: integer = 0;
for num in numbers {
    total = total + num;
}
show total;   // → 15

// Index access
show numbers[0];   // → 1
show names[2];    // → Coq

§ 3

Proof System

The defining feature of the El language is formal verification built directly into the syntax. Proofs are not documentation — they are first-class language constructs verified by the semantic analyzer before any code runs.

Proof Primitives

axiom
Self-evident foundation. Declared true without proof — the bedrock on which theorems are built.
theorem
A proposition asserted to be provable. Must be matched by a proof block bearing its name.
hypothesis
An assumption declared within a proof scope. Held true for the duration of the proof, then discharged at QED.
test
An inline verification step. Checks that a hypothesis or expression holds against established axioms.
QED  Quod Erat Demonstrandum   ∎
Seals and terminates the proof block. Causes the semantic analyzer to verify that all hypotheses have been tested and all required conditions satisfied.

Annotated Example

axiomatic_method.el 
// Lay the axiomatic foundation
axiom identity:
    true === true;

axiom excluded_middle:
    true or !true;

// Declare the theorem
theorem simple_theorem: true;

// Construct the proof
proof simple_theorem {
    hypothesis h1: true;
    test verify_h1: h1: true;
    QED;   // ∎
}
  • axiomSelf-evident truth — no proof required. Added to the axiom pool.
  • axiomSecond axiom extends the logical foundation.
  • theoremDeclares a claim. The semantic analyzer registers it as "unproven".
  • proofOpens a proof scope tied to simple_theorem.
  • hypothesisAssumes h1 = true within this scope.
  • testVerifies h1 holds. Fails with SemanticError if inconsistent.
  • QEDSeals proof. Theorem marked "proven" in symbol table. ∎

Realistic — Three-Valued Logic

The realistic type extends boolean with a third truth value — implementing Kleene's K3 logic for expressing uncertainty, superposition, or probabilistic states.

true— definite truth
false— definite falsity
realistic— indeterminate / uncertain
realistic_demo.el
var confidence: realistic;
var certainty:  realistic = true;
var doubt:      realistic = realistic;

// Kleene conjunction table:
// true ∧ realistic  → realistic
// false ∧ realistic → false
// realistic ∧ realistic → realistic

if confidence {
    show "definitively true";
} else {
    show "uncertain or false";
}

§ 4

Package System

The Bring system is El's module import mechanism — analogous to Python's import or JavaScript's import from. Packages are resolved from the Easier Hub registry and cached locally.

Full package import
bring math_utils;
Import with alias
bring neural_networks as nn;
Named imports (destructured)
bring { Vector, Matrix }
    from math_package;
Scoped source
bring linear_algebra
    from scientific_hub;

Installing Packages

package.bring config + CLI install 
# Generate a sample package config
el download --sample

# Install all packages from config
el download package.bring

# Inspect local cache
el package cache list
el package cache clear

§ 5

CLI Reference

The el command-line tool (v1.0.9) is the primary interface for running El programs, managing packages, and inspecting the cache.

Command Description
el run <file.el> Execute an El source file through the full compiler pipeline
el run -c "code" Compile and run inline El code directly from the shell
el download [config] Install packages declared in a .bring configuration file
el download --sample Generate a sample package.bring file in the current directory
el package cache list List all packages currently stored in the local cache
el package cache clear Remove all packages from the local cache

§ 6

Compiler Pipeline

El is a fully compiled language with a four-stage pipeline implemented in Python. Each stage produces a well-defined artifact consumed by the next — source text → token stream → AST → validated AST → runtime output.

Lexer
→
Parser
→
Semantic Analyzer
→
Interpreter
Stage 01
Lexer
Character-by-character tokenizer. Handles case-insensitive keywords, {{ }} multi-line comments, // line comments, and multi-char operators (===, !=, >=). Emits LexerError on invalid tokens.
Stage 02
Parser
1,152-line recursive descent parser. Builds a complete typed AST for all language constructs — including proof blocks, switch statements, bring declarations, and nested functions. Emits ParserError on syntax violations.
Stage 03
Semantic Analyzer
Traverses the AST for semantic validity: scope & symbol resolution, duplicate identifiers, constant immutability enforcement, theorem–proof linkage verification, and argument count checks. Emits SemanticError.
Stage 04
Interpreter
AST visitor-pattern executor. Runs proof blocks, resolves and executes package imports via the Easier Hub registry, evaluates all built-in functions, and handles runtime errors with structured InterpreterError messages.

Examples

Four complete programs demonstrating the range of El — from classic algorithms to formal axiomatic proofs.

Fibonacci Recursion
fibonacci.el 
ALGORITHM fibonacci_demo {

    function fib(n: integer) {
        if n <= 1 { return n; }
        return fib(n-1) + fib(n-2);
    }

    for i: integer = 0; i < 10; i = i+1 {
        show fib(i);
    }
}

Advanced Proof Multi-hypothesis
proof_demo.el 
axiom p_true: true;
axiom q_true: true;

theorem conjunction: true;
proof conjunction {
    hypothesis hp: true;
    hypothesis hq: true;
    test check_p: hp: true;
    test check_q: hq: true;
    test both: hp and hq: true;
    QED;   // ∎
}
